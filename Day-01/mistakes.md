# Things to Remember
- `input()` always returns a string.
- Use `int()` or `float()` before performing calculations.
- Use `float()` for prices, temperatures, and measurements.
- Use meaningful variable names (`friends` > `frnds`).
- Initialize variables that might be used later (e.g., `coupon_amt = 0`).
- Use `.lower()` to make user input case-insensitive.
- Use f-strings for cleaner output (`f"${total:.2f}"`).
- Think about edge cases (0 friends, negative values, invalid inputs).
- Use comments to organize code into sections.
- Predict the output before running the code.

# Mistakes
- Forgot to use `input()` while asking for the coupon.
- Didn't initialize `coupon_amt`, which could cause an error.
- Used abbreviated variable names instead of descriptive ones.
- Didn't format float values to 2 decimal places initially.
- Didn't consider edge cases in the first version.
